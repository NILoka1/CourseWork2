# Task & Project Management System

Небольшое fullstack-приложение для управления задачами, проектами и командами.
Делал как pet-project, чтобы прокачать архитектуру, работу с ролями и состояние в React.

> Проект в процессе — часть функционала ещё допиливается, деплой будет позже.

---

## Что уже есть

### Задачи

* список задач с фильтрами:

  * все
  * мои
  * backlog
  * выполненные
* статусы:

  ```
  BACKLOG → TODO → IN_PROGRESS → REVIEW → TESTING → DONE
  ```
* действия зависят от роли (кто что может нажать)
* логика действий вынесена отдельно (не размазана по JSX)

---

### Роли и доступ

* роли: `MANAGER`, `TEAM_LEAD`, `DEVELOPER`, `TESTER`
* доступ к страницам ограничен через роутинг (ProtectedRoute)
* UI тоже адаптируется под роль (кнопки, действия)

---

### Команды

* просмотр команды
* редактирование (через форму)
* добавление участников
* привязка проектов

---

### Проекты

* редактирование основных полей:

  * название
  * описание
  * статус
  * прогресс
  * бюджет
  * даты
* нормальная работа с датами (Date → ISO)

---

### Тайм-трекинг

* простой таймер на задаче
* запись времени

---

## Технологии

**Frontend**

* React + TypeScript

* React Router
* Zustand

**Backend**

* Node.js
* Prisma
* PostgreSQL

---

## Как устроено

Старался не сваливать всё в кучу:

```
hooks/       → логика (загрузка, обработка, действия)
components/  → UI
services/    → API
pages/       → страницы
```

### Что из интересного

* сделал `UniversalTasks` — одна точка входа для разных списков задач
* разделил отображение:

  * простой список (done/backlog)
  * полный (с действиями)
* вынес действия задач в конфиг (`getTaskActions`)
* есть обёртка под loading/error (`StatePage`)

---

## Формы

* использую `@mantine/form`
* валидация:

  * на blur
  * ошибки под полями
* форма отделена от серверных данных
* добавил cancel/reset без лишнего состояния

---

## UX мелочи

* sticky блок с кнопками (сохранить / отмена)
* нормальная обработка loading / error
* условный рендер вместо хаоса из if-ов
